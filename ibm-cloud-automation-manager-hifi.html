HIFI

 

 

 

 

 

1. Navigate to Catalog

 

George navigates to the Service Library page to search for the offering that he wants for developing his application.

 

2. Find a Service

 

George browses the Service Library to search for the service offering.

 

Design Story:

As George, I want to easily search and discover services in the catalog.

 

3. Preview a Service

 

George previews the service details.

 

*Sandy can customize this page with multi-media contents.

*Will have Price information in future quarter.

 

 

Design Story:

As George, I want to preview a service, so that I can decide if it meets my requirements, before I order it.

 

3. Preview a Service

 

George previews the Associated Instances details.

 

 

Design Story:

As George, I want to preview a service, so that I can decide if it meets my requirements, before I order it.

 

4. Order a Service

 

Step 1 - Basic Info:

George starts to order the service by inputing Instance Name, Description, Reason for Order, Environment, and Size.

 

 

Design Story:

As George, I want to order a service I want.

 

4. Order a Service

 

Step 2 - Input Parameters:

George fills in values for the parameters exposed from the service.

Then George clicks on the “Order” button to submit the order.

 

*The parameters set will is different based on what Env and Size user selects in Step 1.

 

 

Design Story:

As George, I want to preview a service, so that I can decide if it meets my requirements, before I order it.

 

4. Order a Service

 

George sees a confirmation dialog and then clicks on “Go to Deployed Instances” to check the instance status.

 

 

 

Design Story:

As George, I want to order a service I want.

 

<

5. See the Deployed Instance

 

George sees the ordered service in the Instances view and check the status. It’s still In Progress.

 

After a while, George gets a notification to be notified the ordered instance is approved.

 

 

 

Design Story:

As George, I want to check the approval status of my service order

As George, I want to get a notification to be notified when an order is approved.

 

6. Control a Instance

 

George can take some action of ordered instance.

 

 

Design Story:

As George, I want to control the instances which I ordered.

 

 

 

7. Filter the Instance

 

George can use the navigation on the left or the filter button on the top to filter the instance according to some options.

What's more, George can click the setting button on the top to customize the columns of the instance table.

 

 

8. Manage Instances

 

George can click the instance to expend the column to view the components belong to this instances.

 

 

Design Story:

As George, I want to access the service instances directly from the self service portal.

As George, I want to manage the underlying resources (vm, disks, software, etc) associated to a service instance.

9. Instance Details

 

George can click the name of the instance to view the details of the instance.

 

*George can click “Edit” to update the parameters and then apply the change to running instance.

 

 

 

Design Story:

As George, I want to access the service instances directly from the self service portal.

As George, I want to manage the underlying resources (vm, disks, software, etc) associated to a service instance.

9. Instance Details

 

George can click the resource tap to view the details of the components belong to this instance.

 

 

 

Design Story:

As George, I want to access the service instances directly from the self service portal.

As George, I want to manage the underlying resources (vm, disks, software, etc) associated to a service instance.
